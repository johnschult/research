# Research project workflow

When asked to run a research project in this repo:

1. Create a new folder with a short descriptive name (kebab-case).
2. Inside it, create `notes.md` and append to it as you work — what you tried,
   dead ends, what you learned. This is the running log, not a final report.
3. When the investigation is done, write a `README.md` in the folder: a clear
   report of the question, what you did, what you found, and the conclusion.
4. Update the root `README.md` "## Projects" section with a one-paragraph
   summary linking to the folder.

Final commit should include ONLY:
- The project folder and its notes.md + README.md
- Selected artifacts you produced (scripts, results, small data files)
- Binary files only if under 2MB
- A `git diff` file if you modified an external repo — NOT a full copy of it

Do NOT commit full copies of code or repos you fetched during the investigation.
